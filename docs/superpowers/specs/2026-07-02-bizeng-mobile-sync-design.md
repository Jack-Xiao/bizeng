# BizEng 升级版设计：手机优先 + 云同步 + 输出训练

日期：2026-07-02
状态：已确认（用户批准）

## 背景与目标

用户：四级基础、外企工作多年。痛点全面（会议表达、邮件写作、听力、阅读），
但优先训练**输出**（说/写）——输出能力提升后，听读识别自动受益。

现有资产：单文件 `index.html`（180 个外企高频表达、30 天课表、SM-2 间隔复习、
回想/默写双模式、TTS 发音），数据存 Mac 浏览器 localStorage。

核心问题：用户实际学习场景是**手机碎片时间 + 晚上固定时间**，现有应用只覆盖
Mac；且缺输出训练（造句）与个人语料管道。

目标：6 个月内把 300-500 个外企高频表达从"眼熟"推到"能在邮件和会议里主动用出来"。

## 方案选型

- A（选定）：升级现有单文件应用 → GitHub Pages 托管 + Supabase 云同步 + PWA
- B（放弃）：迁 Anki——同步稳但无法做造句挑战，iOS 客户端收费，放弃定制
- C（放弃）：Mac 照旧 + Claude 手机对话测验——SRS 闭环断裂

## 架构

```
GitHub Pages（免费静态托管，手机/Mac 同一网址）
    │
index.html（保持单文件）+ manifest.json + sw.js（PWA：主屏幕图标、离线可用）
    │
localStorage = 主存储（离线优先，地铁无网可学）
    │
Supabase 免费表 = 同步层（有网时后台自动推/拉）
```

### 云同步设计（Phase 2）

- Supabase 表 `progress`：`sync_code (text, PK)`, `data (jsonb)`, `updated_at (timestamptz)`
- **表对 anon 角色完全关闭（无任何直接 RLS 读写）**。仅暴露两个 security-definer
  数据库函数：`get_progress(code text)` / `put_progress(code text, payload jsonb)`，
  anon 仅有 EXECUTE 权限。函数内按完整 sync_code 精确匹配，不存在列表/枚举路径。
  （注：匿名场景下 RLS 无法表达"必须按完整码查询"，直连表方案不安全，故弃用。）
- 同步码：≥128 位随机（crypto.getRandomValues），展示给用户在另一设备输入一次
- 合并策略：按卡片逐条合并，取"学得更深"的状态（reps 更高者优先；相同则 due 更晚者）；
  用户句子存档与收件箱按时间戳并集去重
- 同步时机：应用启动、每次学习会话结束、手动按钮；失败静默重试，不阻塞学习
- Phase 1 期间跨设备搬进度：用现有导出/导入 JSON 功能

### 迁移

首次在 Mac 打开新网址时，检测旧 `bizEng.v3` localStorage → 自动迁移并上云，进度零丢失。
（同一浏览器同域才可见旧数据；若网址不同域，提供"从旧文件导入 JSON"兜底入口。）

## 三种学习模式

| 模式 | 场景 | 交互 |
|---|---|---|
| 复习模式（现有微调） | 手机碎片 | SRS 到期卡：看中文→回想→翻面自评（忘了/模糊/记得/简单），单手大按钮 |
| 通勤听力模式（新） | 地铁/走路 | TTS 读中文→用户心里说英文→TTS 读英文答案→自评；可纯耳朵操作，自动连播 |
| 造句挑战（新，核心） | 晚上 | 每晚 3 题，两种题型混出：①场景造句（2 题）——给真实工作场景（中文），用当天/近期所学表达写自己的句子；②表达升级（1 题）——给一句平淡/中式英语，要求用目标表达改写成地道商务说法。写完展示范例自查；句子存档；一键复制本周句子（供发给 AI 点评） |

### 掌握标准（核心规则变更）

一个表达只有**被造过句**或**标记为真实工作中用过**才能进入"已掌握"。
仅靠复习次数（reps）最多到"熟练"，不算掌握——复习衡量"记得"，产出衡量"会用"。

## 输入管道（个人语料）

- **快速收件箱**：手机 10 秒完成，只填英文即可保存，状态"待补全"；
  外加一排单击"来源"标签（邮件/会议/文档/Bob/自己想到），可选
- 晚上在 Mac 补全中文/例句（配合 Bob 查词）
- 补全后的收件箱词**优先于**预置课表进入新词队列

## 每周/每月闭环

- **本周实战清单**（Phase 2）：每周一自动从已学（reps>=2）中选 5 词，任务=本周
  用进真实邮件/会议
- **毕业规则（渐进式）**：真实用过 1 次 → 复习间隔大幅拉长（按"简单"调度）；
  真实用过 2 次 → 标记已掌握；之后自评"忘了" → 回炉重新进入复习队列
- 统计页：冻结区（lapse>=3 反复忘的词）、按场景分类的强弱、总进度

## 不做（YAGNI）

- 不做账号/密码系统（同步码足够）
- 不做 AI 自动批改造句（v1 = 范例自查 + 每周复制句子给 Claude 点评）
- 不做 iOS 推送（不可靠；建议手机闹钟/日历提醒，Mac plist 照旧）
- 不做多用户/分享功能

## 错误处理

- 无网：全功能可用（localStorage），同步静默排队
- Supabase 不可达：横幅提示"未同步"，不阻塞学习
- 同步冲突：逐卡合并规则保证不回退进度
- TTS 不可用（个别浏览器）：降级为纯文字模式，隐藏发音按钮

## 测试

- SM-2 调度与合并逻辑：纯函数，node 脚本单测（模拟两设备分叉后合并）
- 迁移逻辑：用现有 v3 存档快照验证
- 手机实机验证：iPhone Safari 加主屏幕、离线打开、TTS 发音、同步往返

## 分阶段实施

**Phase 1（核心学习体验，一次交付）**
1. GitHub Pages 部署（手机可用的前提——iOS 无托管网址则无主屏幕/离线/可靠存储）
2. PWA：manifest + service worker + 手机单手布局
3. 快速收件箱（含来源标签）+ 收件箱词优先进新词队列
4. 造句挑战（场景造句 + 表达升级两种题型）+ 句子存档 + 一键复制
5. 新掌握标准与渐进毕业规则
6. Mac 现有进度：用现有导出/导入 JSON 搬一次家

**Phase 2（跑顺后再做）**
1. Supabase 云同步（security-definer RPC 方案，经已连接的 Supabase MCP 建表建函数）
2. 本周实战清单
3. 统计页增强（冻结区、场景强弱）

排序原则：不让同步/后端工程拖住核心学习体验的升级。

## 交付后的日常使用流

手机主屏幕点开 → 碎片时间刷复习/听力 → 晚上 3 道造句题 →
每周 5 个实战打勾 + 随手收词 → 每月扫一眼统计。
